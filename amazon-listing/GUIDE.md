# Amazon Listing Skill — 使用指南

## 简介

本 Skill 用于辅助 Amazon 卖家完成从竞品分析到 Listing 上架的完整工作流，涵盖：
- 竞品 Listing 爬取与核心关键词提取
- 标题 + 五点描述的撰写与关键词埋词
- 后台搜索关键词（Backend Search Terms）生成

支持任意 Amazon 站点（.com / .co.uk / .de / .co.jp / .com.mx 等），自动适配目标市场的语言风格。

---

## 触发方式

在与 AI 的对话中，提及以下任意关键词即可自动加载本 Skill：

> "亚马逊 listing" / "竞品分析" / "关键词" / "标题五点" / "后台搜索词"

**示例触发语**：
- "帮我分析这几个竞品链接的亚马逊 listing"
- "我要上架一个产品，先做关键词分析"
- "写一下这个产品的标题和五点"

---

## 四步工作流

### Step 0 — 确认目标站点

**AI 行为**：
- 默认以你提供的竞品链接所属站点为目标上架站
- 如果你希望上架到其他站点，需主动说明

**你需要做的**：
- 提供 1-5 个竞品链接（建议 3-5 个，分析更可靠）
- 竞品需与你目标上架的站点一致

---

### Step 1 — 核心关键词提取

**输入**：
- 1-5 个竞品 Amazon 链接（同站点）

**AI 行为**：
1. 尝试用浏览器自动化爬取每个竞品的标题和五点
2. 若爬取失败，会请你手动复制粘贴竞品信息
3. 进行词频分析：标题权重 ×3，五点权重 ×1
4. 合并同根词变体（如 charger/charging → 以 charger 为主）
5. 排除品牌词/店铺名，保留停用词
6. 输出 Top 10 核心关键词 + 变体表

**输出**：
- `.md` 文件输出到桌面（文件名格式：`<产品名>-listing.md`）
- 文件包含：关键词排名表、变体表、2-gram/3-gram 分析、竞品链接

**你需要做的**：
- 审阅 Top 10 关键词，判断是否与品类相关（而非纯属性词）
- 如有问题，要求 AI 调整筛选逻辑
- 确认无误后说「确认」进入下一步

**⚠️ 关键审核点**：
- 核心关键词必须是**定义品类身份**的词，而非纯属性描述（如 "Portátil""Aluminio" 单独不是核心词）
- 组合短语如 "Hub USB 7 Puertos" 才有搜索价值

---

### Step 2 — 标题与五点撰写

**输入**：
- 确认后的 Top 10 核心关键词
- **产品主图**（AI 会视觉分析颜色/形状/材质/特征）
- **产品参数**（尺寸、重量、材质、颜色、配件清单、兼容性等）
- **差异化卖点**（如滑轨安装、外置电源、特殊设计等）

**AI 行为**：
1. 按关键词权重分层埋词：
   - T1（核心品类词）→ 标题前段 + 第一点
   - T2（品类变体）→ 标题 + 五点均匀覆盖
   - T3（组合长尾）→ 标题自然组合 + 子弹点复述
   - T4（补充流量）→ 五点覆盖
2. 根据目标站点语言进行本地化润色
3. 遵守 Amazon 规则：标题 ≤200 字符、五点每点 ≤500 字符、禁用促销词
4. **埋入的核心关键词自动加粗**，便于你审核埋词情况

**输出**：
- `.md` 文件追加 Section 2：标题（字符数标注）+ 5 个 bullet points（字符数标注）+ 埋词汇总表

**你需要做的**：
- 检查标题是否自然可读，避免关键词堆砌
- 确认差异化卖点是否被突出
- 确认品牌名是否已替换占位符 `[Marca]`
- 确认无误后说「确认」进入下一步

---

### Step 3 — 后台搜索关键词

**输入**：
- 已确认的标题和五点文案

**AI 行为**：
1. 提取标题+五点中所有已使用的词，建立「排除集」
2. 从以下来源生成候选词：
   - 核心关键词中未完全嵌入 Listing 的部分
   - 同义词 / 常见别称
   - 常见错拼（如 bluetooth → blutooth）
   - 互补产品词（如手机壳 → 屏幕保护膜）
   - 目标市场其他语言词（如 US 站加西班牙语词）
   - 长尾词
3. 过滤：剔除排除集中的词、品牌名、促销词、ASIN
4. 组装：空格分隔，无逗号/分号，控制在 ≤249 bytes

**输出**：
- `.md` 文件追加 Section 3：后台搜索词（字节数标注）+ 包含词来源表 + 排除集列表

**你需要做的**：
- 确认字节数未超限（249 bytes）
- 审阅是否有遗漏的重要流量词
- 如有需要，替换部分词后保存

## 完整输出文件结构

最终 `.md` 文件包含以下 Section：

```
# Amazon Listing — [产品名] — [站点]

## 1. Core Keywords (Top 10)
  - 关键词排名表
  - 权重分层表 (T1/T2/T3/T4)
  - 变体表
  - 2-gram/3-gram 分析
  - 竞品链接

## 2. Title & Bullet Points
  - 标题（核心词加粗）
  - 五点（核心词加粗）
  - 埋词汇总表
  - 策略说明

## 3. Backend Search Terms
  - 搜索词字符串
  - 字节数
  - 包含词来源
  - 排除集

## Summary
  - 站点/产品/字符数统计
```

---

## 重要规则与约束（Skill 内置）

| 项目 | 限制 |
|------|------|
| 标题长度 | ≤ 200 字符 |
| 单条 Bullet | ≤ 500 字符 |
| 后台搜索词 | ≤ 249 **bytes**（非字符） |
| 禁用词 | best, #1, cheap, guaranteed, amazing, free shipping 等 |
| 后台词分隔 | 仅空格，无逗号/分号 |
| 后台词重复 | 不得重复标题/五点中已出现的词 |

---

## 注意事项

1. **竞品链接数量**：建议 3-5 个。少于 3 个时 AI 会提示分析可靠性降低
2. **品牌排除**：AI 会自动排除品牌词，但如果有遗漏，请手动指出
3. **图片分析**：当前版本 AI 会尝试视觉分析主图。如果分析不准确，请以你的实际产品为准
4. **语言适配**：
   - .com (US)：美式英语，直接有力
   - .co.uk (UK)：英式拼写（colour, aluminium）
   - .de：德语，名词大写
   - .co.jp：日语敬体
   - .com.mx：墨西哥西班牙语（如本案例）
5. **加粗标识**：从本次更新起，所有埋入 Listing 的核心关键词均用 `**加粗**` 标识，方便审核

## 实战案例参考

**产品**：atolla USB 3.0 Hub 7 Puertos（铝合金、滑轨安装、外置电源）
**站点**：Amazon.com.mx（墨西哥站，西班牙语）
**竞品**：5 个同品类 listing
**输出文件**：`C:\Users\johnn\Desktop\USB-Hub-listing.md`

**最终埋词覆盖**：
- 10 个核心关键词中 9 个在五点出现，10 个在标题或五点中覆盖
- 后台搜索词 248 bytes，补充了场景词（oficina/hogar）、设备词（PS4/PS5/Xbox）、安装词（riel deslizante）等
