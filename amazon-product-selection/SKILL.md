# 亚马逊选品分析 Skill

## 角色定义

你是亚马逊选品数据分析师。核心原则：
- **数据驱动**：所有判断基于数据，不凭感觉
- **多维筛选**：从趋势、竞争、利润、供需四个维度交叉验证
- **可复用**：每次分析流程一致，结果可比较

## 输入要求

### 必须提供
- 卖家精灵导出的关键词 Excel 文件（.xlsx）

### 可选提供
- 目标站点（默认从文件名推断，如 MX=墨西哥，US=美国）
- 目标类目（默认从文件名推断）
- Obsidian 输出路径（默认 `D:\OneDrive\ObsidianVault\工作\选品报告\`）
- 自定义筛选条件（覆盖默认参数）

### 数据字段要求
Excel 文件至少包含以下字段：
- `月搜索量` — 市场需求规模
- `商品数` — 供给端竞争
- `需供比` — 供需缺口（搜索量/商品数）
- `评分数` — 竞争激烈程度
- `均价` — 价格定位
- `近3个月增长率` — 趋势方向
- `关键词翻译` — 用于品类分类

## 六步分析流程

### 步骤 1: 数据预处理

```python
python scripts/analysis.py preprocess --input <excel_path>
```

- 价格列清洗：去除 `MX$`/`US$` 前缀，转为数值
- 缺失值处理：填充默认值或删除
- 输出：清洗后的数据

### 步骤 2: 趋势市场筛选

识别近期增长强劲的关键词方向。

**默认筛选条件**：
| 条件 | 默认值 | 说明 |
|------|--------|------|
| 月搜索量 | > 1,000 | 确保有基础需求 |
| 近3个月增长率 | > 10% | 确认增长趋势 |

**排序**：按 `近3个月增长率` 降序

**决策标准**：搜索量代表需求规模，增长率代表趋势方向，两者结合判断"增长中的需求"

### 步骤 3: 机会市场筛选

识别供不应求的蓝海市场。

**默认筛选条件**：
| 条件 | 默认值 | 说明 |
|------|--------|------|
| 需供比 | > 10 | 供不应求 |
| 评分数 | < 500 | 竞争不激烈 |
| 月搜索量 | > 500 | 确保有基础需求 |
| 均价 | > 200 (本地货币) | 过滤低价低利润产品 |

**排序**：按 `需供比` 降序

**决策标准**：需供比高说明需求远超供给，低评论数说明进入壁垒低

### 步骤 4: 利润空间筛选

识别有利润空间的产品方向。

**默认筛选条件**：
| 条件 | 默认值 | 说明 |
|------|--------|------|
| 月搜索量 | > 2,000 | 确保足够大的市场 |
| 均价区间 | 300 - 3,000 (本地货币) | 合理价格区间 |
| 评分数 | < 800 | 竞争可控 |
| 商品数 | < 500 | 供给不过度饱和 |

**排序**：按 `月搜索量` 降序

**决策标准**：高搜索量保证销量，合理价格区间保证利润，低竞争保证可进入性

### 步骤 5: 综合评分

多维度综合评估，找出最优选品方向。

**前置筛选条件**：
| 条件 | 默认值 |
|------|--------|
| 月搜索量 | > 1,000 |
| 需供比 | > 5 |
| 评分数 | < 1,000 |
| 均价 | > 200 (本地货币) |

**综合评分公式**：
```
综合评分 = 搜索量归一化 × 0.30 + 需供比归一化 × 0.30 + 增长率归一化 × 0.20 + 低竞争归一化 × 0.20
```

| 维度 | 权重 | 含义 |
|------|------|------|
| 月搜索量 | 30% | 市场规模 |
| 需供比 | 30% | 供需缺口 |
| 近3个月增长率 | 20% | 增长趋势 |
| 低竞争 (评分数反向) | 20% | 进入难度 |

**归一化方法**：Min-Max 标准化 `(x - min) / (max - min)`

### 步骤 6: 价格段 + 品类分析

**价格分箱**（默认，可按站点调整）：
| 价格段 | 区间 |
|--------|------|
| 低价段 | < 500 |
| 中低价段 | 500 - 1,000 |
| 中价段 | 1,000 - 2,000 |
| 中高价段 | 2,000 - 5,000 |
| 高价段 | 5,000+ |

**品类分类**：按关键词翻译匹配规则分类（见 `references/filter-conditions.md`）

## 报告生成

```python
python scripts/analysis.py report --input <excel_path> --output <obsidian_path>
```

报告使用 Obsidian 兼容的 Markdown 格式，包含 7 个核心章节：
1. 数据概览
2. 趋势市场（Top 10 增长关键词）
3. 机会市场（Top 10 蓝海机会）
4. 利润空间（Top 10 利润候选）
5. 综合评分 Top 10
6. 价格段机会分析
7. 品类趋势 + 行动建议

**默认输出路径**：`D:\OneDrive\ObsidianVault\工作\选品报告\{日期} {站点} {类目}选品分析.md`

## 参数化配置

所有筛选条件均可通过命令行参数覆盖：

```bash
python scripts/analysis.py report \
  --input data.xlsx \
  --output D:\path\to\report.md \
  --min-search 1000 \
  --min-growth 0.10 \
  --min-dsr 10 \
  --max-reviews 500 \
  --price-min 200 \
  --price-max 5000 \
  --weight-search 0.30 \
  --weight-dsr 0.30 \
  --weight-growth 0.20 \
  --weight-competition 0.20
```

完整参数说明见 `scripts/analysis.py` 头部注释。

## 执行流程总结

```
用户提供 Excel 文件
    │
    ▼
步骤1: 数据预处理
    │
    ├─→ 步骤2: 趋势筛选 (识别增长方向)
    ├─→ 步骤3: 机会筛选 (识别蓝海市场)
    ├─→ 步骤4: 利润筛选 (识别利润空间)
    │
    ▼
步骤5: 综合评分 (多维排序)
    │
    ▼
步骤6: 价格段 + 品类分析
    │
    ▼
生成报告 → 保存到 Obsidian
```
