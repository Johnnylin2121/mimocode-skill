# 每日复盘工作流

## 核心原则（必须严格遵守）

1. **时效性优先**：任何阶段开始前，必须先拉取实时行情数据（指数、个股、板块资金流向），确保所有判断基于最新信息。
2. **预期-验证闭环**：每日复盘提出的假设，次日早盘验证，闭环反馈。
3. **数据驱动**：所有判断必须有具体数据支撑，数值比较要精确到百分比或绝对值。
4. **连贯性追溯**：盘后复盘必须关联前两个交易日复盘文件（缺失时降级为当日数据+前两日大盘数据辅助）。
5. **交易计划分离**：复盘只分析市场，不关联持仓；交易计划的审查独立进行，仅检查前提和失效条件，不自动下单。
6. **盘中纪律**：盘中只执行已制定的计划，不临时修改交易计划；修改只能在盘后审查时进行。
7. **三日窗口**：复盘主要基于今日 + 前两个交易日，归档时仅保留最近三个交易日的复盘文件。
8. **时间校验**：每个阶段开始前必须通过 PowerShell 获取当前日期，确认今日、前一交易日、前二交易日，避免日期错误。

## 工作流三阶段

### 阶段一：盘前准备（用户触发，通常 9:00 前）

**前置动作**：
1. 执行时间校验（`Get-Date -Format "yyyy-MM-dd dddd"`），确定今日、前日、前前日。
2. 强制拉取实时指数数据（上证、沪深300、创业板指）及关键板块资金流向。

**输入**：
- 前两个交易日复盘文件（路径 `D:\OneDrive\ObsidianVault\交易体系\每日复盘\`，缺失时降级处理）
- 当日财经早读 `D:\OneDrive\ObsidianVault\交易体系\财经早读\YYYY-MM-DD-财经早读.md`（缺失立即报错并终止）
- 所有状态为"⏳ 等待入场"或"🟢 持仓中"的交易计划（路径 `D:\OneDrive\ObsidianVault\交易体系\交易计划\`）

**步骤**：
1. **读取复盘文件**：读取前两个交易日的复盘文件。若某日文件不存在，继续向前追溯；若最终只找到不足两个，则改为使用当日数据 + 拉取前两个交易日的大盘指数作为辅助（降级模式）。
2. **提取关键信息**：从前日复盘文件中提取"明日关注点"和"可验证假设"；从当日早读文件中提取核心新闻与政策动态。
3. **交易计划审查**（自动执行）：
   - 逐条检查所有未完结交易计划：
     * 计算入场条件的数值完成度（如价格距离突破位还需涨 X%、量能距离条件还差 X 亿）。
     * 若任一条件满足程度 ≥ 80%，主动提醒用户："有介入机会，是否制定/执行交易计划？"并给出精确的数值对比。
     * 检查持仓计划的前提是否仍成立（对比当前市场方向与决策依据），**若前提不成立，自动将计划状态改为"❌ 已失效"并移入 `存档\` 子目录**，无需人工确认。
     * 若失效条件已被触发，同样自动标记失效并归档。
   - 输出审查结论（如下格式）：
```markdown
### 交易计划审查
- [计划名称]：⏳ 等待入场 — 继续等待
  - 入场条件：价格 14.3 距突破位 15.0 还需上涨 4.9%；量能 2.35 万亿距 2.5 万亿差 6.0%，未达阈值。
  - 前提成立：是（市场方向未变）
  - 操作建议：继续等待

- [另一计划]：❌ 已失效 — 前提不成立，已自动归档
```

4. **制定观察清单**：
   - 基于实时数据、复盘结论、早读信息，生成三层观察清单：
     * 第一层：市场整体量能（成交额判断标准：>2.8万亿=确立；2.5-2.8万亿=试探；<2.5万亿=萎缩）
     * 第二层：板块龙头量能验证（各方向龙头个股+判断标准）
     * 第三层：科技分化信号（关键标的+判断标准）
   - 包含明确的决策树（如：放量>2.8万亿→确认；缩量→观望）。
   - 写入文件 `D:\OneDrive\ObsidianVault\交易体系\盘前预测\YYYY-MM-DD 盘前预测.md`。

---

### 阶段二：盘中验证（用户触发，建议 10:00 / 13:00）

**前置动作**：
1. 执行时间校验。
2. 拉取实时数据（指数、观察标的个股、板块资金流向）。

**步骤**：
1. **验证前日假设**：读取前日复盘文件中的"可验证假设"，用当前实时数据逐条验证，标记 ✅/❌。
2. **观察清单核对**：逐条比对盘前制定的观察清单，记录当前状态。
3. **交易计划跟踪**：
   - 检查所有"⏳ 等待入场"计划的入场条件接近程度，给出数值比较。
   - 若入场条件满足度 ≥ 80%，主动提醒："有介入机会，是否执行交易计划？"
   - 若任何持仓计划触发止损线，立即提醒："[计划名] 已触发止损条件，建议执行。"
   - 若市场方向出现重大变化可能影响持仓计划前提，立即提醒审查。
4. **写入验证结果**：将验证内容追加到**前日复盘文件**的"今日验证"章节（不是当日复盘！）。

---

### 阶段三：盘后复盘（用户触发，15:00 后）

**前置动作**：
1. 时间校验，确定今日、前日、前前日。
2. 拉取当日完整交易数据（收盘价、成交额、资金流向等）。

**输入**：
- 当日所有盘中验证结果。
- 前两个交易日复盘文件（必须追溯，缺失时降级处理）。
- 所有未完结交易计划。

**步骤**：
1. **创建当日复盘报告**（路径：`D:\OneDrive\ObsidianVault\交易体系\每日复盘\YYYY-MM-DD 每日复盘.md`）。
   - 报告必须包含以下章节（严格顺序）：
     1. 核心摘要（前日结论、今日验证、核心判断一句话）
     2. 前日复盘回顾（复述前两日核心结论，并验证其假设）
     3. 大盘与量能（指数表格、量能分析）
     4. 资金动向（板块、个股资金流入流出 TOP10）
     5. 核心驱动（3-5 条宏观/产业驱动）
     6. 板块表现（强弱板块对比）
     7. 连贯性验证（与前日判断对比：正确/偏差/遗漏）
     8. 轮动逻辑（资金路径）
     9. 明日关注点（主线、标的、判断标准）
     10. 操作建议
     11. 思维引导（必问的 5 个问题）
     12. 今日归因（驱动力识别、因果链条、持续性判断）
     13. 可验证假设（至少一个，包含验证标准和比对组）
2. **交易计划审查**（盘后自动执行）：
   - 对照当日复盘结论，检查每个未完结计划的决策依据是否仍成立。
   - 检查失效条件是否触发。
   - 更新计划的审查记录：
     * 前提成立 → 记录"审查通过，继续等待"
     * 前提动摇 → 记录"审查警告：[具体变化]，建议关注"
     * 前提不成立 → 自动标记失效，移入存档
   - 若发现新介入机会，提醒用户制定新交易计划，但不自动创建。
3. **归档清理**：
   - 检查 `每日复盘` 目录，将早于前前日的复盘文件移入 `存档\` 子目录。
   - 确保主目录仅保留今日、前日、前前日三个交易日的复盘文件。

## 数据获取方式

### 东方财富API

**指数行情**：
```powershell
$url = "https://push2.eastmoney.com/api/qt/ulist.np/get?fltt=2&fields=f2,f3,f4,f6,f12,f14&secids=1.000001,0.399001,0.399006"
```

**个股行情**：
```powershell
# 市场代码格式：1.600000(沪市) / 0.000001(深市)
$url = "https://push2.eastmoney.com/api/qt/ulist.np/get?fltt=2&fields=f2,f3,f4,f5,f6,f7,f8,f10,f12,f14,f15,f16,f17,f18&secids={市场代码}"
```

**板块资金流向**：
```powershell
$url = "https://push2.eastmoney.com/api/qt/clist/get?pn=1&pz=20&po=1&np=1&fltt=2&invt=2&fid=f62&fs=m:90+t:2&fields=f12,f14,f62,f184,f3"
```

**个股资金流向**：
```powershell
$url = "https://push2.eastmoney.com/api/qt/ulist.np/get?fltt=2&fields=f62,f184,f12,f14&secids={市场代码}"
```

**API字段映射**：
| 字段 | 含义 |
|------|------|
| f2 | 最新价 |
| f3 | 涨跌幅(%) |
| f4 | 涨跌额 |
| f5 | 成交量(手) |
| f6 | 成交额(元) |
| f7 | 振幅(%) |
| f8 | 换手率(%) |
| f10 | 量比 |
| f12 | 股票代码 |
| f14 | 股票名称 |
| f15 | 最高价 |
| f16 | 最低价 |
| f17 | 开盘价 |
| f18 | 昨收价 |
| f62 | 主力净流入(元) |
| f184 | 主力净流入占比(%) |

## 错误处理

- API 调用失败：重试一次，仍失败则跳过该数据项并在报告中标注。
- 财经早读文件缺失：立即报错并终止盘前流程。
- 前两日复盘文件缺失：降级为当日数据 + 前两日大盘辅助，继续执行。
- 非交易日：提示并退出。

## 常见错误

1. **时间混淆**：Notion API 返回 UTC 时间，需 +8 小时转换为东八区
2. **早盘验证写错位置**：应写入前日复盘的"今日验证"，不是当日复盘
3. **图片占位符**：markitdown 转换会生成 `![](data:image/jpeg;base64...)`，需清理
4. **JSON 编码**：PowerShell 的 ConvertTo-Json 对 Notion API 会出错，需用 UTF-8 文件方式
5. **复盘中关联交易计划**：复盘只分析市场，不关联持仓。交易计划的审查在复盘完成后独立进行
6. **盘中临时修改交易计划**：交易计划一旦制定，盘中只执行不修改。修改只能在盘后审查时进行
7. **交易计划制定无人工参与**：MiMo 可以提醒有介入机会，但交易计划必须经用户确认后才创建
8. **前日复盘只追溯一天**：必须关联前两个交易日的复盘文件，如某日无复盘文件则继续向前追溯
9. **日期错误**：每个阶段开始前必须先获取当前日期，确认今日、前一交易日、前二交易日
